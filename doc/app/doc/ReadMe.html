    RapiData

    The Gateway to Data.gov.in

    RapiData makes searching for datasets fast, efficient and fun. It consists of two parts:

    • RapiData API
    • Front-end UI

    RapiData API:

    The API is built as a standard Ruby on Rails application
    API Specification:
    Query Link: /datasets/retreive_from_terms.xml
    Query Parameter: terms -> it a string consisting of terms separated by spaces
    Response format: XML
    Response structure: 

        <rapidata>
      <datasets>
        <dataset>
            <title>dataset.title</title>
            <description>dataset.description</description>
            <link>dataset.link_to_data.gov.in_page</link>
            <department>dataset.department</department>
            <priority>dataset.priority</priority>
            <download_link>dataset.download_link</download_link>
         </dataset>
           .
           .
           .      
      </datasets>
      <terms>
        <term>term_1</term>
        <term>term_2</term>
           .
           .
           .
      </terms>
    </rapidata>

    The datasets are used to construct search listings and the terms are used to construct the tag clouds.

    RapiData UI:

    The UI is based on Responsive Design Principles using jQuery and Bootstrap.

    Plan for sustenance:

    Currently the app uses only 400 datasets. To keep this app updated, we need the list of all datasets with their titles, descriptions, departments,links to their data.gov.in page and their download links on a regular basis from the people at data.dov.in.

    Apart from that, we will resolve the bugs that may crop up from time to time.
